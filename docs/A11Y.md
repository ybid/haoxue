# 无障碍与儿童友好设计（A11Y）

> 适配儿童认知与运动特点 + WCAG 2.1 AA 标准

---

## 一、设计原则

| 原则 | 含义 |
|---|---|
| **可感知** | 信息以多种感官通道呈现（视觉 + 听觉 + 触觉） |
| **可操作** | 足够大的点击区、足够的反应时间、容错友好 |
| **可理解** | 语言简单、反馈直接、不依赖隐性知识 |
| **健壮** | 兼容辅助技术与不同设备 |

**双重用户**：

- 主用户是 6\~8 岁儿童（认知、运动能力发展中）
- 次用户是家长（可能带老花、需快速浏览）

---

## 二、儿童认知适配

### 2.1 视觉通道

| 维度 | 规范 | 理由 |
|---|---|---|
| 字号下限 | 按钮 36\~48px、算式 80\~120px | 孩子读字仍在发展中 |
| 字重 | 600\~700 | 加粗提高可读性 |
| 字间距 | `letter-spacing: 0.02em` | 略宽便于辨识 |
| 行高 | `line-height: 1.5` | 减少拥挤感 |
| 颜色对比 | 主文字 12:1、按钮文字 4.6:1 | WCAG AA + AAA |
| 形状辨识 | 数字 + 图形结合（如 3 + 🍎×5） | 跨视觉型学习者友好 |

### 2.2 听觉通道

- **答对音**：上行三音符，明亮（C5-E5-G5）
- **答错音**：低音短促，不刺耳（C4 衰减）
- **通关音**：上行琶音，仪式感
- **音效开关**：家长面板可控，避免部分儿童敏感
- **音量**：不超过 65dB，避免长时间使用损伤听力

### 2.3 触觉通道（iOS 触觉反馈）

iPad Safari PWA 暂不直接支持 Haptic Feedback API（需原生应用）。

**替代方案**：

- 按钮按下视觉反馈（缩放 0.96）
- 错误抖动（视觉震动）
- 答对涟漪动画

### 2.4 时间通道

- 答对停顿可配置（800\~1500ms，默认 1000ms）
- 不限时（核心模式）
- 计时挑战可调时长（30/60/90 秒）

## 三、运动能力适配

### 3.1 点击区

**儿童手指点击区 ≥ 64×64 pt**（Apple HIG 推荐 44pt，但儿童建议更大）。

```css
.btn {
  min-width: 120px;
  min-height: 64px;
  padding: 16px 24px;
}

.option {
  min-width: 100px;
  min-height: 100px;
  font-size: 48px;
}
```

### 3.2 间距

- 选项按钮之间留白 ≥ 16px
- 主屏关卡卡片间距 ≥ 24px
- 避免误触

### 3.3 防误触

```html
<!-- viewport 设置 -->
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no, viewport-fit=cover">
```

```css
* {
  -webkit-tap-highlight-color: transparent;
  touch-action: manipulation;
}
```

### 3.4 触屏容错

- 错误点击**不换题**，孩子可继续尝试
- 滑动手势**不触发**任何危险操作（重置、退出）
- 长按 1 秒内不触发关键操作

## 四、家长无障碍

### 4.1 视觉

- 家长面板字号默认 16px，可调至 20px
- 7 天练习时长用柱状图 + 数字双轨展示
- 错误率用百分比 + 进度条

### 4.2 操作

- 危险操作（重置、删除错题）需**二次确认**
- 二次确认需输入"删除"等关键词
- 误操作可一键撤销最近一次删除（仅限错题清理）

---

## 五、视觉无障碍（WCAG 2.1 AA）

### 5.1 对比度

| 元素 | 前景 | 背景 | 对比度 |
|---|---|---|---|
| 主文字 | `#2C3E50` | `#FFFBF5` | 12.6 : 1 ✅ AAA |
| 次文字 | `#7F8C8D` | `#FFFBF5` | 4.7 : 1 ✅ AA |
| 按钮文字 | `#FFFFFF` | `#5BA8E0` | 4.6 : 1 ✅ AA |
| 成功文字 | `#2D8C3C` | `#E8F8EC` | 5.5 : 1 ✅ AA |
| 错误文字 | `#C03A2B` | `#FDECE9` | 5.7 : 1 ✅ AA |
| 锁图标 | `#95A5A6` | `#F0EAD9` | 3.0 : 1 ⚠️ |

> 锁图标仅起装饰作用，不承载关键信息（文字"未解锁"）。

### 5.2 不依赖颜色

- 答对：**绿色 + ✓ 图标**
- 答错：**红色 + ✗ 图标 + 抖动**
- 通关：**金色 + 星星动画 + 文字"已通过"**

色盲用户（红绿色盲）可凭图标识别。

### 5.3 焦点指示

```css
:focus-visible {
  outline: 3px solid #5BA8E0;
  outline-offset: 2px;
}
```

键盘用户可见焦点轮廓。

### 5.4 文字缩放

```css
html {
  font-size: 16px;
}

/* 家长面板"大"字号档 */
html.font-large {
  font-size: 20px;
}
```

iOS Safari 最大可缩放至 200%（虽然设置了 user-scalable=no，但可通过字号档覆盖）。

## 六、运动无障碍

### 6.1 减弱动效

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

家长面板提供「动效」开关：

```js
if (settings.motion === false) {
  document.documentElement.classList.add('no-motion');
}
```

```css
.no-motion * {
  animation: none !important;
  transition: none !important;
}
```

### 6.2 不依赖精确计时

- 答对后停顿用 `setTimeout`，无强同步依赖
- 计时挑战结束时**不强制立即终止**，允许最后一题完成判定
- 操作可中断（计时挑战中途退出不丢失历史最高分）

---

## 七、听觉无障碍

### 7.1 视觉冗余

- 每个音效都有对应的视觉反馈（绿色高亮 / 红色抖动 / 星星动画）
- 不依赖声音传达关键信息（不通过声音告诉孩子"答错"）

### 7.2 静音支持

- 默认开启音效
- 家长面板可关闭所有音效
- 关闭后所有交互仍正常工作

### 7.3 音量控制

- 主音量使用系统音量
- 不提供应用内音量滑块（避免复杂度）

---

## 八、认知无障碍

### 8.1 语言

- 全程简体中文
- 简单句式，避免复杂术语
- 算式表达用阿拉伯数字 + 数学符号（`+ - × ÷ = ?`），不用汉字"加 减 乘 除"

### 8.2 一致性

- 相同操作在不同页面行为一致
- 主按钮位置一致（右下或居中）
- 返回箭头始终在左上

### 8.3 帮助与提示

- 首次进入家长面板显示 1\~2 句使用引导
- 错题本空状态显示鼓励语
- 通关时显示下一步提示

### 8.4 错误恢复

- 错点可继续尝试（不换题）
- 切关卡后当前关卡进度自动保存
- 重置需二次确认 + 关键词

## 九、辅助技术兼容

### 9.1 屏幕阅读器

虽然目标用户为儿童，但家长/视障家长仍需辅助技术。

```html
<button class="option" aria-label="选项 A，数字 8">
  8
</button>
```

### 9.2 ARIA 标注

```html
<!-- 进度条 -->
<div class="progress" role="progressbar"
     aria-valuenow="4" aria-valuemin="0" aria-valuemax="8"
     aria-label="连击进度 4 / 8">
  <div class="progress-bar" style="width: 50%"></div>
</div>

<!-- 答对反馈 -->
<div class="toast" role="status" aria-live="polite">
  太棒了！
</div>

<!-- 错误反馈 -->
<div class="toast" role="status" aria-live="assertive">
  再看看哦~
</div>
```

`aria-live="polite"` 用于一般提示；`assertive` 用于错误需要立刻被朗读。

### 9.3 跳过链接

```html
<a href="#main-content" class="skip-link">跳到主要内容</a>
```

```css
.skip-link {
  position: absolute;
  left: -9999px;
}
.skip-link:focus {
  left: 16px;
  top: 16px;
  background: #5BA8E0;
  color: white;
  padding: 8px 16px;
  z-index: 999;
}
```

## 十、键盘无障碍

| 按键 | 功能 | 范围 |
|---|---|---|
| `Tab` | 下一个交互元素 | 全局 |
| `Shift+Tab` | 上一个交互元素 | 全局 |
| `1\~4` | 选择对应选项 | 训练页 / 挑战页 |
| `Space` / `Enter` | 确认 / 下一题 | 训练页 |
| `Esc` | 返回 / 取消 | 全局 |

**焦点顺序**：自上而下、自左而右，符合视觉阅读顺序。

**焦点陷阱**：模态框打开时，焦点循环在模态框内。

## 十一、色盲模式

### 11.1 红绿色盲

- 答对：不仅绿色，伴随 ✓ 图标 + 轻微缩放动画
- 答错：不仅红色，伴随 ✗ 图标 + 抖动 + 提示文字

### 11.2 蓝黄色盲

- 主色蓝色（`#5BA8E0`）与黄色（`#FFC857`）在蓝黄色盲下区分度尚可
- 进度条使用渐变 + 数字文字双轨

---

## 十二、儿童特定注意事项

### 12.1 防止过度刺激

- 不使用闪烁效果（避免光敏性癫痫风险，闪烁频率 > 3Hz 禁用）
- 庆祝动画 ≤ 3 秒
- 不使用高分贝或刺耳音效

### 12.2 防止成瘾

- 无连续使用奖励（如"连续 5 分钟奖励"）
- 无积分累计、排行榜
- 家长面板可设置"每日时长上限"（提示而非强制）

### 12.3 隐私

- 不询问姓名、不收集个人信息
- 不显示其他孩子的数据
- 不接入社交功能
