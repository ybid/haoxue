# 产品需求文档（PRD）

> 口算无痛训练工具 · iPad 网页版 · 二年级适配

---

## 一、产品愿景

让 6\~8 岁的孩子通过**无压力、纯选择题**的方式，**真正掌握**二年级口算每一道题，建立对数学学习的**安全感与掌控感**。

**不做的事**：不搞刷题排名、不绑定家长焦虑、不引入物质激励、不上传数据。

**核心信念**：不是"做对 N 题就过关"，而是"**每一道题都真正掌握**才算过关"。错题进入错集优先重出，直到孩子把它答对为止。

## 三、核心场景

### 场景 1：日常练习（精熟判定）

1. 孩子从主屏点击当前关卡
2. 屏幕中央出现算式，下方 4 个选项按钮
3. 孩子点击选项：
   - 正确：✅ 高亮 + 音效 + 该题**标记为已掌握** → 出下一题
   - 错误：🔴 按钮抖动 + 提示语 → 该题**进入错集** → 孩子可继续在剩余选项中尝试
4. 下一题优先从**错集**抽取，错集空再随机抽新题
5. 当关卡内**所有题目都标记为已掌握** → 通关动画 + 解锁下一关

### 场景 2：错集复习（关键流程）

1. 错集是关卡的"待掌握列表"
2. 错集中的题比新题**优先级更高**（优先抽取）
3. 错集中的题答对 1 次（首次正确）→ 标记掌握 → 从错集移除
4. 错集中的题答错 → 仍在错集，继续优先重出
5. 错集为空 = 本关全部掌握

### 场景 3：超时未答处理

1. 题目长时间未作答（默认无超时，但家长可设置）
2. 系统判定"未掌握"，该题进入错集
3. 算式切换到下一题

> **默认无超时**，因为核心定位是"无痛训练"。超时为可选的家长设置项（默认关闭）。

### 场景 4：家长查看学情

1. 主屏角落小齿轮 ⚙️ 图标
2. 连点 3 次 / 长按 2 秒进入家长面板
3. 查看各关卡"已掌握 / 总题数"
4. 查看错题集内容
5. 7 天练习时长、首次正确率等数据
6. 可选操作：导出错题 / 修改参数 / 重置全部

## 四、精熟判定算法（核心）

### 4.1 三种状态

每道题（按算式识别）对每个关卡有三种状态：

| 状态 | 含义 |
|---|---|
| **未出现** | 该题未被抽到过 |
| **待掌握（错集）** | 孩子答错过，等待重出 |
| **已掌握** | 孩子首次答对 |

### 4.2 状态转换

```
未出现 ──[抽到+首次正确]──► 已掌握
   │
   └────────────[抽到+首次错误]──► 待掌握 ──[重出+首次正确]──► 已掌握
                                       │
                                       └────[重出+首次错误]──► 仍待掌握
```

### 4.3 通关条件

**两个条件同时满足**：

1. ✅ **错集为空**（没有 pending 题）
2. ✅ **当前 session 内所有抽到的题都已掌握**

> 不再使用"最小掌握集合"硬性要求，孩子实际做了多少题就掌握多少题，错题全部清除即通关。

### 4.4 超时判定（核心防卡机制）

**全题统一超时**——任何题（新题、错集题、冷却恢复题）都限时作答：

| 关卡类型 | 默认超时 | 原因 |
|---|---|---|
| **一位数关卡**（关卡 1） | **5 秒** | 一位数运算简单，5 秒足够 |
| **两位数及以上**（关卡 2\~6） | **10 秒** | 两位数需要更长时间思考 |

**超时反馈**：

- 剩余 3 秒时：选项按钮闪烁提示
- 超时未答：自动判为错误，该题进入错集
- 反馈语："没关系的，下次再来~"（不指责）

**家长可调**：超时时长家长面板可改（1 位数 3\~10 秒 / 2 位数 5\~30 秒）。

**保证全覆盖**：所有题目都参与超时判定，避免题目卡住不答。

### 4.5 出题策略（关键算法）

```js
function nextQuestion(level) {
  // 1. 优先：错集未掌握
  const errorSet = getErrorSet(level.id);
  if (errorSet.length > 0) {
    return pickRandom(errorSet);
  }

  // 2. 次优：未覆盖过的题（保证题库覆盖）
  const unseen = getUnseenQuestions(level);
  if (unseen.length > 0) {
    return pickRandom(unseen);
  }

  // 3. 兜底：从全部题库随机（这种题会自动进入错集被优先重出）
  return generateRandom(level);
}
```

### 4.6 关键设计点

| 设计点 | 选择 | 理由 |
|---|---|---|
| 错集优先级 | 最高 | 确保错题一定会被重做 |
| 已掌握不再出 | 是 | 避免无效重复，保护孩子时间 |
| 通关条件 | "错集空 + 当前 session 全部已掌握" | 简单直接，孩子做完就通关 |
| 超时判定 | **默认开启**（1 位 5s / 2 位 10s） | 保证全覆盖，避免题目卡住 |
| 超时覆盖范围 | **全题**（错集/冷却/新题都限时） | 防止任何题目卡住 |
| 倒计时提示 | 剩 3 秒闪烁 | 不突然判超时，给孩子反应时间 |
| 冷却机制 | 同题连续错 5 次 → 冷却 5 分钟 | 防挫败，避免无限循环 |

## 五、关卡体系

| ID | 名称 | 内容 | 操作数范围 | 默认超时 | 解锁条件 |
|---|---|---|---|---|---|
| 1 | 一位数加法 | 0\~9 + 0\~9 | 全表 100 题 | **5 秒** | 默认 |
| 2 | 两位数加法 | 10\~99 + 10\~99 | 全表 | **10 秒** | 第 1 关通过 |
| 3 | 混合加减 | 两位数 ± 一位数 | 全表 | **10 秒** | 第 2 关通过 |
| 4 | 表内乘法 | 1\~9 × 1\~9 | 81 题 | **10 秒** | 第 3 关通过 |
| 5 | 表内除法 | 整除，商 1\~9 | 81 题 | **10 秒** | 第 4 关通过 |
| 6 | 综合运算 | + - × ÷ 混合 | 各关卡并集 | **10 秒** | 第 5 关通过 |

**超时规则**：

- 关卡 1（一位数）：5 秒
- 关卡 2\~6（两位数及以上）：10 秒
- 家长面板可调（关卡 1：3\~10 秒；其他：5\~30 秒）

**已解锁关卡可随时回退练习**，进度独立保存。

## 七、需求评审

### ★ 方案亮点

- ★ **算法根基扎实**：精熟学习理论的真正实现——每题都要掌握
- ★ **错集优先**：错题不被遗忘，自动重做直到掌握
- ★ **超时全题覆盖**：避免任何题目卡住，保证流程顺畅
- ★ **分关卡超时**：1 位 5 秒、2 位 10 秒，匹配孩子认知节奏
- ★ **倒计时提示**：剩 3 秒闪烁，不突然判超时
- ★ **冷却防挫败**：同题连续错 5 次暂停 5 分钟
- ★ **情感安全设计**：超时无指责、错误可修正
- ★ **隐私优先 + 零成本**：纯本地 + PWA + 单文件

### ⚠️ 潜在风险

| 编号 | 风险点 | 说明 |
|---|---|---|
| R-01 | **通关时间不确定** | 取决于错题多少，可能通关较慢 |
| R-02 | **错集越积越多** | 冷却机制缓解但未根除 |
| R-03 | **两位数加法**题库过大 | 10\~99 = 8100 种组合，全掌握不现实 |
| R-04 | **超时判定**对慢思考孩子不友好 | 倒计时提示 + 家长可调缓解 |
| R-05 | **错题本与错集关系** | 已明确：错题本是错集的全量历史 |

### 💡 优化建议

#### M0 必须优化

1. **进度可视化**："已掌握 X 道"实时展示
2. **错集展示**：让孩子看到"还有 N 道待掌握"
3. **倒计时 UI**：剩 3 秒选项闪烁
4. **超时反馈语**：温和不指责
5. **超时时长可调**：家长面板支持
6. **两位数加法策略**：默认随机抽取（无最小集合硬性要求）

#### M1 应该优化

7. **错题本 = 错集的全量历史**（含已通过的）
8. **家长面板区分"当前错集"与"历史错题"**
9. **成就"掌握 N 道题"**
